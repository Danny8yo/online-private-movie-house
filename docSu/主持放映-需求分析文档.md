# 主持放映-需求分析

## 1. 从用例筛选候选类

### 1.1 候选类提取

- 从用例"主持放映"描述中识别候选类：

| 候选类                  | 提取来源               | 筛选问题                         |
|-------------------------|-----------------------|----------------------------------|
| VideoStatus（视频状态） | “系统更新视频播放状态” | 需要持久化视频状态吗？           |
| OperationLog（操作日志）| “系统记录操作日志”     | 需要记录管理员操作吗？           |
| Admin（管理员）（存在疑问） | “管理员请求播放/暂停” | 管理员没有持久化存储信息是类吗？ |
| PlayControlUI（管理员操作界面） | “管理员通过界面控制” | 需要专门的交互界面吗？           |
| PlayController（放映控制） | “系统处理播放控制请求” | 需要协调多个对象吗？             |



## 2.分析边界、控制、实体类

### 2.1 分类标准

- 边界类：处理系统与外部参与者的交互
- 控制类：协调用例流程，管理业务逻辑
- 实体类：代表持久化业务数据

### 2.2 最终分类结果
| 类别 | 类名称 | 理由 |
|------|--------|------|
| 边界类 | PlayControlUI | 管理员与系统交互的界面，提供播放控制功能 |
| 控制类 | PlayController | 协调播放控制的业务逻辑，处理管理员请求 |
| 实体类 | VideoStatus | 存储视频播放状态（进度、播放状态、倍速、视频源、字幕源） |
| 实体类 | OperationLogs | 记录管理员操作的操作日志集合 |
| 实体类 | OperationLog | （单个操作日志） |


- 为什么不包含管理员：管理员在该用例中仅作为外部参与者，触发系统功能，并没有需要持久化存储的信息，不作为实体类，不符合实体类、边界类或控制类的定义

## 3.职责分配与对象协作分析

- 为确保用例中的每个步骤都有明确的对象负责，我们对对象间的协作关系进行分析：

### 3.1 播放/暂停（取自主持放映的一部分）场景的对象协作

| 步骤 | 涉及的类 | 协作的类 | 职责说明 |
|------|---------|---------|---------|
| 1. 管理员请求播放/暂停 | PlayControlUI | - | 接收管理员（参与者）的界面操作输入 |
| 2. 系统处理播放控制请求 | PlayController | PlayControlUI | 协调业务流程，处理控制请求 |
| 3. 更新视频状态 | VideoStatus | PlayController | 改变播放状态并记录变更 |
| 4. 记录操作日志 | OperationLogs | PlayController | 创建并保存操作日志记录 |
| 5. 显示操作结果 | PlayControlUI | - | 向管理员（参与者）反馈操作结果 |

### 4.2类图推导

### 4.2.1 识别类的属性和操作


### PlayControlUI (边界类)

**职责：** 提供管理员操作界面，接收外部参与者输入，显示操作结果  
**操作：**
- `+startInterface()`: 启动播放控制界面
- `+togglePlayback()`: 向控制类请求处理播放/暂停请求
- `+setSpeed()`: 向控制类请求设置播放速度
- `+setPlaybackProgress()`: 向控制类请求设置播放进度
- `+setVideoSources()`: 向控制类请求设置视频源
- `+setSubtitle()`: 向控制类请求设置字幕
- `+showResult()`: 显示操作结果

### PlayController (控制类)
**职责：** 协调播放控制业务逻辑，处理各种播放控制请求  
**操作：**
- `+togglePlayback()`: 协调处理播放/暂停逻辑
- `+setSpeed()`: 协调处理播放速度调整
- `+setPlaybackProgress()`: 协调处理播放进度调整
- `+setVideoSource()`: 协调处理视频源切换
- `+setSubtitle()`: 协调处理字幕设置
- `+logOperation()`: 协调记录操作日志

### VideoStatus (实体类)
**职责：** 存储和维护视频播放状态信息  
**属性：**
- `-playbackProgress`: 当前播放进度
- `-playback: boolean`: 播放状态
- `-speed: float`: 播放速度
- `-videoSource`: 视频源
- `-subTitleSource`: 当前字幕源

**操作：**
- `+togglePlayback()`: 切换播放/暂停状态
- `+setSpeed()`: 设置播放速度
- `+setPlaybackProgress()`: 设置播放进度
- `+setVideoSource()`: 设置视频源
- `+setSubtitle()`: 设置字幕

### OperationLogs (实体类)
**职责：** 管理操作日志的集合，提供日志记录功能  
**属性：**
- `-logs`: 操作日志列表

**操作：**
- `+logOperation()`: 记录操作日志

### OperationLog (实体类)
**职责：** 记录单个操作的详细信息  
**属性：**
- `-operationType`: 操作类型
- `-operationTime`: 操作时间

### 4.2.2 识别类之间的关系
| 关系类型 | 类A | 类B | 多重性 | 描述 |
|----------|-----|-----|--------|------|
| 关联 | PlayControlUI | PlayController | 1对1 | UI向控制器发送控制请求 |
| 关联 | PlayController | VideoStatus | 1对1 | 控制器协调更新视频状态 |
| 关联 | PlayController | OperationLogs | 1对1 | 控制器协调记录操作日志 |
| 组合 | OperationLogs | OperationLog | 1对多 | OperationLogs包含多个OperationLog对象 |
